# Security policy

## Supported versions

Only the latest published Packy release receives security fixes. Upgrade to the
latest release before reporting a problem that may already be corrected.

## Reporting a vulnerability

Do not disclose vulnerability details, credentials, recovery material, or a
working exploit in a public issue or pull request.

Use GitHub's private **Report a vulnerability** form on the repository Security
tab when it is available. If private reporting is unavailable, contact
[`@yersonargotev`](https://github.com/yersonargotev) through a private channel
listed on that profile and share only enough public information to establish a
private reporting path.

Include the affected Packy version, platform, impact, reproduction conditions,
and any proposed remediation. Do not test against systems or accounts you do
not own or have explicit permission to assess.

The maintainer will acknowledge a usable private report, coordinate validation
and remediation, and publish an advisory when disclosure is safe. Response and
release timing depends on severity and the availability of a verified fix.
